# PowerGrid-AlexZ
Powergrid Project
# Part 1: The Presentation
Your group will program the Powergrid board game. The first thing you must do is give a presentation to
the other groups and the teacher about how you will program this project. First, the teacher will
determine how many groups there will be for the class and then choose 1 student to be the project lead
of their soon-to-be-created group. The teams will be chosen using a snake draft. The project leader will
be the person in charge of the project and should make sure their team is doing equal parts of the project.
Once the teams are filled, then moving day occurs where all groups are assigned a working area in the
class. At this point, the teacher will show how the game is played. Depending on the size of groups, this
could be done in a day or a couple of days.
The new group will create an agreed upon code of conduct and expectations document which should
cover what the expectations are in the group. How the group will decide who receives a 5, 4, 3, 2, and 1
score for the week. This is like a social contract and will be due for a quiz grade on the Monday after
groups are assigned from the group lead.
The students will produce, for a test grade and due March 20th, a written presentation with a verbal
presentation on how they will program the project. They should not program this project at this time.
Among the items the teacher will be looking for in this presentation are:
1. A physical presentations for the teacher, presented in a professional business way. This
presentation should be given like a technical paper with a cover sheet and a brief outline to start and then
should go into the details of the project. You must also upload the presentation for all other groups to see
and follow along.
2. A detailed time outline with due dates and assigned jobs with deadlines, meetings, and milestones
present including but not limited to the creation of the document that is being presented on that same
day.
3. A UML of the complete planned program, the more detailed, the better. There should be no
programming at this time. This should include, but not limited to, a game state class for functionality and
independent graphics classes that will be used to visualize the game. The graphics class should only be
calling the game state and not incorporating the game functionality into the graphics.
4. A mockup of the expected game user interface screens explaining what every button does.
5. A series of planned test data to verify that the program will work and to be used in debugging,
this includes possible scenarios that could occur during the game. The team should write test data for
every little piece of detail in the code to ensure the code will work.
6. A discussion of foreseeable problems with the program and how the team plans to overcome
them.
The presentation will also be given in front of the class to explain their plan. The team will be graded by
the other teams and the teacher. Each team will keep every team’s proposal so that they can use them to
judge the final project. At the conclusion, each team will rank all the other teams in their class, and the
teacher will rank all the teams from all the classes. Those rankings along with the score given by the
teacher and their peers will contribute to determining a final grade for the project. The team’s peers will
also write an assessment of each of their teammates on Friday of each week of the project. These
performance reviews will be used in determining the individual’s minor score for the presentation at the
end of the six weeks.
# Part 2: The Game
A youTube video: Power Grid Board Game - How to Play. By Stella and Tarrant teaches how to play the
game. I have provided you with images and a pdf of the rulebook which can be found on Canvas. You
may create your own images if you choose.
It is highly recommended, but not required, that each team buys a copy of the Powergrid game. We will
only be programming the base game. The game currently retails for around forty five dollars on Amazon.
Playing the game physically amongst each other and your family and friends will allow the programmers
to understand the game better. With a better understanding of the game, the programmer will be able
to program the game better. I will have a copy of Powergrid available in the room as well.
The teams are expected to program a 3 player game at least and to use the Germany map instead of the
United States map.
